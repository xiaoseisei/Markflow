## 一、产品定位

### 1.1 一句话定位

> 面向开发者与知识工作者的**本地优先 Markdown 编辑器**，兼具 Obsidian 的文件管理能力与 Typora 的沉浸式编辑体验。

### 1.2 核心价值主张

| 维度 | 价值描述 |
|------|---------|
| **轻量** | 安装包 < 15MB，启动时间 < 2s |
| **专注** | 专为 Markdown 写作场景设计，无冗余功能 |
| **本地优先** | 文件存储在本地，用户完全掌控数据 |
| **开箱即用** | 无需配置，打开即用，支持直接管理文件夹 |

### 1.3 目标用户

| 用户群体 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| **开发者** | 写技术文档、README、开发日记 | 日常工作配合 IDE 使用 |
| **学生/研究者** | 做笔记、整理知识体系 | 课堂记录、论文草稿 |
| **内容创作者** | 写博客草稿、脚本 | 需要导出为多种格式 |
| **项目经理** | 写 PRD、会议纪要 | 多文件管理，团队协作 |

## 二、竞品分析

| 产品 | 优势 | 劣势 | 我们的机会点 |
|------|------|------|------------|
| **Typora** | 所见即所得、极简美观 | 付费 $14.99、无文件树、闭源 | 开源免费 + 文件管理 |
| **Obsidian** | 插件生态丰富、双链知识图谱 | 学习曲线陡、渲染性能较慢 | 更轻量、更快、开箱即用 |
| **VS Code** | 功能强大、可高度扩展 | 非专用写作工具、体验割裂 | 专注写作场景 |
| **Notable** | 界面简洁、标签系统 | 停止更新、导出能力弱 | 持续迭代 + 强导出 |
| **Mark Text** | 开源、实时预览 | 维护不活跃、Bug 较多 | 稳定维护 + 现代技术栈 |

## 三、功能需求（MoSCoW 优先级）

### 🔴 Must Have — MVP 核心功能（v1.0）

#### F1 · 文件侧边栏

- 树形文件夹结构渲染，支持展开/折叠
- 操作：新建文件、新建文件夹、重命名（双击/F2）、删除（Del + 确认弹窗）
- 右键上下文菜单（New File / New Folder / Rename / Delete / Copy Path）
- 文件图标区分：文件夹 📁 / Markdown 📄 / 其他文件 📎
- 当前激活文件高亮显示
- 支持打开本地文件夹作为工作区（记忆上次工作区）

#### F2 · Markdown 编辑器

- 代码语法高亮（支持 50+ 编程语言代码块）
- 行号显示，当前行高亮
- 自动补全：括号、引号、Markdown 语法（**、`、[]()）
- 撤销/重做（无限次，Ctrl+Z / Ctrl+Y）
- 查找替换（Ctrl+F，支持正则）
- 字体大小可调（Ctrl++ / Ctrl+-）

#### F3 · 实时预览

- 三种视图模式切换：
  - `split`：编辑与预览左右分屏（默认）
  - `editor`：纯编辑模式（无干扰写作）
  - `preview`：纯预览模式（阅读模式）
- 编辑区与预览区滚动同步
- 预览支持：标题、表格、代码块（带语言标识）、任务列表、数学公式（KaTeX）、图片

#### F4 · 文件操作

- 自动保存（编辑停止后 500ms 触发，防抖）
- 手动保存：Ctrl+S（保存后 Toast 提示"已保存"）
- 标题栏未保存状态指示（● 圆点前缀）
- 关闭未保存标签时弹出确认对话框
- 支持打开单个 .md 文件（不限于工作区）

#### F5 · 导出 PDF

- 菜单项 + 快捷键 Ctrl+Shift+P 触发
- 导出配置：纸张大小（A4/Letter）、方向（纵向/横向）
- 保留完整 Markdown 样式（代码高亮、表格样式）
- 默认调用系统打印对话框（MVP 阶段）

---

### 🟡 Should Have — 1.x 迭代功能

#### F6 · 多格式导出

| 格式 | 说明 |
|------|------|
| **HTML** | 内联 CSS，可直接在浏览器打开的完整 HTML 文件 |
| **DOCX** | 保留 H1-H6 标题层级、粗体、斜体、代码块 |
| **Markdown** | 原始 .md 文件另存为 |

#### F7 · 全局搜索

- 跨所有工作区文件的全文搜索（Ctrl+Shift+F）
- 结果列表显示：文件名 + 匹配行 + 上下文预览
- 点击结果跳转到对应文件的对应行，关键词高亮

#### F8 · 主题系统

- 亮色/暗色模式切换（跟随系统 / 手动切换）
- 代码块主题：GitHub Light / One Dark / Dracula / Nord
- 自定义编辑器字体（系统字体列表）

#### F9 · 大纲面板

- 右侧浮动面板，自动提取 H1-H6 标题
- 点击标题平滑滚动到对应位置
- 当前可见标题高亮

### 🟢 Could Have — 2.x 远期规划

- **双向链接**：`[[wiki-link]]` 语法支持，链接图谱视图
- **Git 版本历史**：查看文件修改历史，diff 对比
- **自定义 CSS 主题**：用户上传 CSS 文件自定义预览样式

## 四、用户体验流程

### 4.1 首次启动流程

```
首次启动
  └─> 欢迎界面（简洁）
        ├─> [打开文件夹] → 选择本地文件夹 → 加载文件树 → 新建第一个文件
        └─> [打开文件] → 选择单个 .md 文件 → 直接进入编辑
```

### 4.2 日常编辑流程

```
启动应用
  └─> 自动恢复上次工作区 + 上次打开的标签页
        └─> 光标定位到上次编辑位置
              └─> 编辑内容
                    ├─> 500ms 无操作 → 自动保存 → 标题栏更新
                    └─> Ctrl+S → 立即保存 → Toast "已保存 ✓"
```

### 4.3 导出 PDF 流程

```
Ctrl+Shift+P / 菜单 File > Export > PDF
  └─> 弹出导出配置对话框
        ├─> 选择纸张大小、方向
        ├─> 预览缩略图
        └─> [导出] → 系统打印对话框 → 选择"保存为 PDF"
```

### 4.4 多文件切换流程

```
侧边栏双击文件 → 在新标签页打开
  ├─> 文件已在标签页中 → 切换到该标签（不重复打开）
  └─> 文件未打开 → 新建标签页，读取文件内容，更新编辑器
```

## 五、非功能性需求

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| **启动时间** | < 2 秒（冷启动） | 从双击图标到编辑器可交互 |
| **文件加载** | < 100ms（< 5MB 文件） | 双击文件到内容显示 |
| **自动保存延迟** | 500ms 防抖后执行 | 计时器验证 |
| **内存占用** | < 150MB（空载） | Task Manager 监测 |
| **安装包大小** | < 15MB | 打包后文件大小 |
| **编辑器流畅度** | 60fps，无明显卡顿 | Chrome DevTools Performance |

## 六、边界条件与异常处理

| 场景 | 处理方式 |
|------|---------|
| 文件被外部程序修改 | 检测到变更 → 弹出提示"文件已被外部修改，是否重新加载？" |
| 文件写入失败（磁盘满/无权限） | 错误 Toast + 保留内存中内容，不丢失用户数据 |
| 打开超大文件（> 10MB） | 警告提示 + 禁用实时预览（仅显示编辑器） |
| 文件夹被删除 | 侧边栏标记为灰色，标签页显示"文件已移动或删除" |
| 关闭有未保存更改的标签 | 确认对话框："是否保存对 xxx.md 的更改？[保存] [不保存] [取消]" |
